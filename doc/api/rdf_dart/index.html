rdf_dart

rdf_dart library

Support for working with RDF (Resource Description Framework) data in Dart.

This library provides classes and utilities for representing and manipulating RDF data structures, including:

  • Terms: IRITerm, BlankNode, Literal, RdfTerm, TripleTerm
  • Triples: Triple
  • IRIs:: IRI
  • Graphs: Graph
  • Datasets: Dataset
  • Exceptions: RDFException, InvalidTermException, DatatypeNotFoundException, InvalidLexicalFormException, InvalidLanguageTagException, LiteralConstraintException

Terms

RDF data is built from terms, which can be one of three types:

  • IRIs (Internationalized Resource Identifiers): Used to name things (resources). See IRI.
  • Blank Nodes: Anonymous nodes within an RDF graph, used to represent things without a specific IRI. See BlankNode.
  • Literals: Data values, like strings, numbers, and dates. See Literal.

The RdfTerm class is an abstract base class for all term types. The TermType enum is used to represent the type of an RDF term.

Triples

RDF data is expressed as triples, which are statements about resources. Each triple consists of a subject, a predicate, and an object. See Triple.

Graphs

An RDF graph is a collection of RDF triples. See Graph.

Datasets

An RDF dataset is a collection of RDF graphs. It contains a default graph and zero or more named graphs. See Dataset.

Usage

To use this library, add rdf_dart as a dependency in your pubspec.yaml file.

dependencies:
  rdf_dart: ^1.0.0 # Or the latest version

Then, import the library in your Dart code:

import 'package:rdf_dart/rdf_dart.dart';

void main() {
  // Create an IRI
  final iri = IRI('http://example.com/resource');
  print(iri);

  // Create a Blank Node
  final blankNode = BlankNode();
  print(blankNode);

  // Create a Literal
  final literal = Literal("Hello", IRI('http://www.w3.org/2001/XMLSchema#string'));
  print(literal);

  // Create a Triple
  final triple = Triple(iri, iri, literal);
  print(triple);
}

Classes

BlankNode
Represents a blank node in an RDF graph.
Dataset
Represents an RDF Dataset, which is a collection of RDF graphs.
Graph
Represents an RDF graph, which is a collection of RDF triples.
IRI
Represents an Internationalized Resource Identifier (IRI) according to RFC 3987.
IRITerm
Represents an Internationalized Resource Identifier (IRI).
Literal
Represents a literal value in an RDF graph.
RdfTerm
The abstract base class for all RDF terms.
SubjectTerm
Abstract marker class for RDF terms that can function as the subject of an RDF Triple.
Triple
Represents an RDF triple, which is a statement about a resource.
TripleTerm
Represents an RDF Triple used as an RDF Term (RDF 1.2).
XSDDuration
A class which represents a duration in terms of its constituent components.

Enums

CardinalityFacet
Every value space has a specific number of members. This number can be characterized as finite or infinite. (Currently there are no datatypes with infinite value spaces larger than countable.)
OrderedFacet
For some datatypes, this specifies an order relation for their value spaces the ordered facet reflects this.
TermType
Represents the type of an RDF term according to RDF 1.2.
XMLDataType
This represents all of the supported XML data types as specified in RDF 1.2

Exceptions / Errors

DatatypeNotFoundException
Exception thrown when a required datatype IRI is not found in the registry or is not supported by the library.
InvalidLanguageTagException
Exception thrown when a language tag string is not well-formed according to BCP 47 syntax.
InvalidLexicalFormException
Exception thrown when a literal's lexical form is invalid according to the rules of its specified datatype.
InvalidTermException
Base class for exceptions related to invalid RDF terms.
LiteralConstraintException
Exception thrown for violations of RDF literal constraints.
RDFException
Base class for exceptions specific to the RDF Dart library.
